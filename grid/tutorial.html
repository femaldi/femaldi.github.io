OPERATOR FIELD MANUAL

Welcome, Operator. You have gained access to the system. Your objective is to navigate the network grid and acquire target data. This manual contains the operational parameters of your interface. Study it carefully. Failure to comply with system protocols will result in instability and termination.

CONTROLS

  • ARROW KEYS - Move the Operator one tile. Each move consumes 1 tick.
  • SPACEBAR - Wait in place. Consumes 1 tick.
  • MOUSE CLICK - Select a tile on the grid when in TARGETING mode.
  • ESCAPE - Cancel the current command selection process.
  • [UNDO] - Removes the last command from the Execution Queue.
  • [RESTART LEVEL] - Aborts the current attempt and restarts the level from the beginning.
  • [BACK TO LEVEL SELECT] - Disconnects and returns to the node selection screen.

CORE MECHANICS

The Tick System & Execution Queue

The system operates in discrete time units called 'ticks'. Every action you take, whether moving or waiting, advances the system by one tick.

  1. Queue Commands: Select a command from the side panel. Some commands require you to select target tiles on the grid. Queuing a command consumes a corresponding amount of your BUFFER.
  2. Enter Execution Mode: Move your Operator onto an Execution Point (X).
  3. Process the Queue: While in EXECUTION MODE, every subsequent move or wait action (which still costs 1 tick) will process the next command in your queue.

System Instability & Reset

Executing certain commands (like DELETE or COPY) destabilizes the local node. When this happens, a SYSTEM_RESET timer will appear. If the timer reaches zero, the level's layout will reset to its original state. Your Operator, queued commands, and buffer usage will remain unchanged. Be warned: if the system reset causes you to be inside a wall or on a Sentinel's spawn point, you will be crushed, resulting in a full level restart.

Target Validation & Mobile Entities

The system is volatile. A command's target must be in a valid state at the moment of execution. If a mobile entity (like a SENTINEL) moves onto a tile you targeted for deletion or a copy *destination*, the command will fizzle and fail, consuming the buffer but having no effect. This is a critical security feature.

COMMANDS

DELETE

Targets 1 tile. Deletes the highest-priority object on that tile. Triggers a System Reset.

COPY

Targets 2 tiles: a SOURCE then a DESTINATION. The state of the source tile is "snapshotted" when you select it. The command then attempts to replicate that snapshot at the destination. The destination tile is completely cleared before the copy occurs. Triggers a System Reset.

NOTE: Because the copy destination is validated at execution time, a SENTINEL moving onto the destination tile will cause the command to fail. Plan accordingly.

WAIT

No target. Pauses the execution of the queue for a set number of ticks. Does NOT trigger a System Reset. Useful for timing your actions around moving Sentinels.

BLOCK IDENTIFIERS

◆
Operator (You)
Your cursor in the system. Use it to navigate and initiate command execution.
#
Wall
Blocks all movement. Expensive to modify (x2 multiplier).
█
Hard Wall
Blocks all movement. Very expensive to modify (x8 multiplier).
|
Door / Gate
Blocks movement when enabled. Can be disabled by a Button.
||
Secured Door
A more expensive Door (x4 multiplier). Blocks movement when enabled. Can be disabled by a Button.
O
Sentinel
A mobile security program. Moves one tile per tick. Collision with a Sentinel results in a full level restart. Sentinels are affected by Arrow Tiles and can press Buttons.
»
Arrow Tile
Forces any entity (Operator or Sentinel) that stands on it to move in the indicated direction on the next tick, overriding any other movement input. Can be enabled or disabled by Buttons.
B
Button
A pressure plate that interacts with other blocks. When an entity (Operator or Sentinel) stands on it, the Button sends an "ON" signal to all linked blocks.
  • When a linked block receives an "ON" signal from ALL of its connected buttons, its state is flipped (e.g., an enabled Door becomes disabled).
  • When an entity steps off a Button, the "ON" signal is turned off. The linked block will remain in its activated state for 1 extra tick before reverting to its normal state.
Buttons can be targeted by COPY and DELETE commands.
●
Data Node
The objective. Move onto this tile to acquire the data and complete the level. Data nodes have the highest command multiplier (x15).
X
Execution Point
A stable system node. Moving onto this tile begins processing your queued commands.
